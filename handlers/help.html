API Reference

Event Booking System — REST API documentation. All protected routes require a JWT Bearer token.

Base URL  http://localhost:8080/api
Authentication
POST /auth/register Create an account ▶

Register a new user. The role field must be either organizer or customer.

Public

Request Body

{ "name": "Alice", "email": "alice@example.com", "password": "secret123", "role": "organizer" }

Response · 201

{ "id": 1, "name": "Alice", "email": "alice@example.com", "role": "organizer", "created_at": "..." }
POST /auth/login Get a JWT token ▶

Authenticate and receive a JWT token valid for 24 hours. Include this token as Authorization: Bearer <token> on all protected routes.

Public

Request Body

{ "email": "alice@example.com", "password": "secret123" }

Response · 200

{ "token": "eyJhbGci...", "user": { "id": 1, "role": "organizer", ... } }
Events
GET /events List all events ▶

Returns all events ordered by date ascending.

Public

Response · 200

{ "data": [ { "id": 1, "title": "Go Summit", "date": "...", "capacity": 100, ... } ], "total": 1 }
GET /events/{id} Get event detail ▶

Fetch a single event by its ID.

Public

Response · 200

{ "data": { "id": 1, "title": "Go Summit", "location": "Bangalore", "price": 499.99, ... } }
POST /events Create a new event ▶

Create an event. Only the authenticated organizer can manage their own events. Date must be in RFC3339 format.

JWT RequiredOrganizer only

Request Body

{ "title": "Go Summit 2026", "description": "All things Go", "location": "Bangalore", "date": "2026-12-01T18:00:00Z", "capacity": 100, "price": 499.99 }

Response · 201

{ "data": { "id": 1, "organizer_id": 1, "title": "Go Summit 2026", ... } }
PUT /events/{id} Update an event ▶

Update all fields of an event. Must be the organizer who created it. Automatically notifies all customers with confirmed bookings in the background.

JWT RequiredOrganizer only

Request Body

{ "title": "Go Summit 2026 — Updated", "description": "...", "location": "Mumbai", "date": "2026-12-05T18:00:00Z", "capacity": 150, "price": 599.99 }
DELETE /events/{id} Delete an event ▶

Permanently deletes an event. Must be the owning organizer.

JWT RequiredOrganizer only

Response · 200

{ "message": "event deleted" }
GET /events/{id}/bookings List event's bookings ▶

Returns all confirmed bookings for an event, including customer name and email. Must be the owning organizer.

JWT RequiredOrganizer only

Response · 200

{ "data": [ { "id": 1, "customer_name": "Bob", "customer_email": "bob@example.com", "tickets": 2, ... } ], "total": 1 }
Bookings
POST /events/{id}/book Book tickets ▶

Book one or more tickets for an event. The booking is rejected if remaining capacity is insufficient. A confirmation is logged in the background on success.

JWT RequiredCustomer only

Request Body

{ "tickets": 2 }

Response · 201

{ "data": { "id": 1, "event_id": 1, "tickets": 2, "status": "confirmed", ... } }

Response · 409 (capacity exceeded)

{ "error": { "message": "not enough capacity", "requested": 10, "remaining": 3 } }
GET /bookings My bookings ▶

Returns all bookings made by the authenticated customer, with event title and date.

JWT RequiredCustomer only

Response · 200

{ "data": [ { "id": 1, "event_title": "Go Summit", "event_date": "...", "tickets": 2, "status": "confirmed" } ], "total": 1 }
DELETE /bookings/{id} Cancel a booking ▶

Cancels a booking by setting its status to cancelled. Must be the customer who made the booking.

JWT RequiredCustomer only

Response · 200

{ "message": "booking cancelled" }
